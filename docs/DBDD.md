## 1. Scope

### 1.1 Identification

This document describes the database design for the GenAI IDP POC deployed into AWS account `198082850288` (us-east-1). The system uses Amazon DynamoDB for structured data and Amazon S3 for document/object storage.

### 1.2 System Overview

The IDP POC stores extraction configuration (document classes, field definitions, prompts) and document processing state in DynamoDB. Uploaded documents, processing artifacts, and extraction results are stored in S3.

## 3. DynamoDB Tables

### 3.1 Configuration Table

- **Purpose:** Stores extraction configuration — document class definitions, field specifications, and LLM prompts for classification and extraction.
- **Table name:** `idp-configuration-table-0378stu2`
- **Capacity mode:** Provisioned (5 RCU / 5 WCU)
- **Encryption:** KMS customer-managed key (`304d492f-7b7f-4bc5-849f-2bff67f3735f`)

| Attribute | Type | Key | Description |
|-----------|------|-----|-------------|
| PK | S | Partition Key | Configuration item identifier |
| SK | S | Sort Key | Version or attribute type |

- **Access pattern:** Read by AppSync resolvers (UI configuration display) and by pipeline Lambda functions (classification/extraction prompts). Written by AppSync mutations (user saves configuration).

### 3.2 Tracking Table

- **Purpose:** Tracks document processing status through the pipeline stages.
- **Table name:** `idp-tracking-table-0378stu2`
- **Capacity mode:** Provisioned (5 RCU / 5 WCU)
- **Encryption:** KMS customer-managed key (`304d492f-7b7f-4bc5-849f-2bff67f3735f`)
- **TTL:** `ExpiresAfter` attribute (7-day retention)

| Attribute | Type | Key | Description |
|-----------|------|-----|-------------|
| PK | S | Partition Key | Document ID |
| SK | S | Sort Key | Timestamp or stage |
- **Access pattern:** Written by each pipeline Lambda (status updates). Read by AppSync resolvers (UI status display, subscriptions). Queried by user to see processing history.
- **Retention:** `data_tracking_retention_days = 7` (TTL-based cleanup)

### 3.3 Capacity Constraints

| Constraint | Vocareum Limit | Expected Usage |
|------------|---------------|----------------|
| On-Demand Read Capacity | 1,000 RCU | <10 RCU (single-user, few documents) |
| On-Demand Write Capacity | 1,000 WCU | <10 WCU (single-user, few documents) |

## 4. S3 Buckets

### 4.1 Input Bucket

- **Purpose:** Stores uploaded documents (PDFs, images) pending processing.
- **Bucket name:** `poc-idp-input-571e2c0p`
- **Encryption:** SSE-S3 (AES256) — pre-created via CLI
- **Access:** Pre-signed PUT URLs generated by AppSync resolver for browser upload. GetObject by OCR Lambda function.
- **Event trigger:** S3 PutObject → EventBridge → SQS → Step Functions (initiates processing pipeline)
- **Note:** Created via CLI (not Terraform) to avoid SCP Object Lock issue.

### 4.2 Output Bucket

- **Purpose:** Stores extraction results (structured JSON), OCR output, and intermediate processing artifacts.
- **Bucket name:** `poc-idp-output-571e2c0p`
- **Encryption:** SSE-S3 (AES256) — pre-created via CLI
- **Access:** PutObject by pipeline Lambda functions. GetObject by AppSync resolvers for results display.

### 4.3 Working Bucket

- **Purpose:** Temporary working files during document processing.
- **Bucket name:** `poc-idp-working-571e2c0p`
- **Encryption:** SSE-S3 (AES256) — pre-created via CLI

### 4.4 Assets Bucket

- **Purpose:** Lambda layer zips, CodeBuild artifacts, deployment assets.
- **Bucket name:** `poc-idp-assets-571e2c0p`
- **Encryption:** SSE-S3 (AES256) — pre-created via CLI
- **Note:** Contains locally-built Lambda layer zips (CodeBuild is blocked by Vocareum).

### 4.5 Web App Bucket — NOT DEPLOYED

- **Reason:** CloudFront `CreateDistribution` is SCP-blocked. Web UI module disabled (`web_ui.enabled = false`). API-only deployment validated via CLI and local React dev server.

### 4.4 Storage Constraints

| Constraint | Vocareum Limit | Expected Usage |
|------------|---------------|----------------|
| Total S3 size (all buckets) | 10 GB | <100 MB (web assets + few test documents + results) |

## 5. KMS Key

- **Purpose:** Encrypts S3 objects and DynamoDB tables at rest.
- **Key ID:** `304d492f-7b7f-4bc5-849f-2bff67f3735f`
- **Key ARN:** `arn:aws:kms:us-east-1:198082850288:key/304d492f-7b7f-4bc5-849f-2bff67f3735f`
- **Alias:** `alias/poc-idp-571e2c0p`
- **Key type:** Symmetric, customer-managed
- **Cost:** $1/month (prorated <$0.01 for <1hr deployment)
- **Deletion:** Scheduled for deletion during `terraform destroy` (7-day pending deletion window; no charges during pending deletion).
